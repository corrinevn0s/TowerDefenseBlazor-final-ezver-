# Tower Defense — простая браузерная игра на Blazor WebAssembly

Простая tower defense игра, написанная на **C# + Blazor WebAssembly** (.NET 8/9).  
Без внешних игровых движков, спрайтов и сложной графики — всё рисуется на `<canvas>` через JavaScript Interop.

### Основные возможности

- 3 типа врагов: быстрые (мало HP), обычные, танки (много HP, медленно)
- 3 типа башен: лук (дешёвая, быстрая), огонь (сильный урон), яд (большая дальность)
- Фиксированный извилистый путь
- Ручной запуск волн (кнопка «Начать волну» активна только когда карта очищена)
- Подсчёт золота, жизней, волн
- Экран поражения с кнопкой «Начать заново»
- Отключение кнопок при game over
- Затемнение экрана при поражении
- Таблица лидеров

### Технологии

- Blazor WebAssembly (InteractiveWebAssemblyRenderMode)
- Чистый `<canvas>` + JS Interop (без Blazor.Extensions.Canvas)
- C# для всей игровой логики (враги, башни, снаряды, волны)
- База данных для сохранения результатов

### Как запустить

1. Открыть проект в Visual Studio 2022/2025
2. Установить .NET 8.0 или новее
3. Запустить (F5 или Ctrl+F5)
4. Перейти по адресу: `https://localhost:7002/game`

### Крупные ошибки и уроки на пути разработки

| № | Проблема | Причина | Как решили | Урок |
|---|--------|--------|------------|------|
| 1 | BECanvas не реагировал на `@onmousedown` | Blazor.Extensions.Canvas не поддерживает стандартные события Blazor | Перешли на чистый `<canvas>` + JS Interop | Старая библиотека (2021) плохо совместима с .NET 8+ |
| 2 | `BECanvas` вообще не рендерился (нет <canvas> в DOM) | Конфликт пространств имён и неправильные `@using` | Удалили библиотеку, перешли на нативный canvas | Пакеты старше .NET 6 часто ломаются на событиях |
| 3 | Много красных ошибок «неоднозначность» (ambiguous reference) | Дубликат класса `GameState` или конфликт имён | Явно указали `TowerDefenseBlazor.Models.GameState` | Всегда проверяй поиск по решению на дубликаты классов |
| 4 | Оверлей «Игра окончена» не появлялся | `position: absolute` внутри ограниченного контейнера → улетал за экран | Перенесли на `position: fixed; inset: 0; display: flex` | Центрирование absolute работает относительно ближайшего positioned родителя |
| 5 | После game over canvas «замерзал», оверлей не отображался | `StateHasChanged()` не вызывался после остановки цикла | Добавили `StateHasChanged()` + последний `Draw()` | При остановке игрового цикла нужно явно обновлять UI |
| 6 | Кнопка «Начать волну» спамилась | Нет защиты от повторного нажатия | Добавили `WaveInProgress` + проверку `!game.Enemies.Any()` | Всегда защищай кнопки от многократного вызова |
| 7 | Проблема: В LeaderboardService.cs использовались полные URL|Конфликт с BaseAddress| Замена на относительные пути| Всегда стоит проверять и искать конфликтующие файлы
### Итог

Проект показал, насколько мощным может быть Blazor даже для простых игр без Unity и др.
